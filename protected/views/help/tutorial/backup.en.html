Lesson: Database backups

  • Why

    • There is no computer today in existance that cannot eventually loose your data you have so carefully put into it.  Therefore, unfortunately, you must reguarly "backup" your data so that you can always get it again if the computer should happen to loose it.    You may loose a few days work but you wont loose years of work anyway.

    Then in the case of this program, there is an extra benifit that goes along with this backup process.  You can backup one company, and then resore that company to another name.  That way you could for example, backup and restore your company to another name, and then "play" with the new company with for example new data to see a prognosis of what could happen.  Or you can backup one company and send that file to your accountant who could resore it to his computer under a special name and then manipulate the data and printout whatever is needed.
     

  • How

    • Simply go to the Advanced->backup database, restore, backup company, restore company.  You will need to choose a location and file name to backup to.  And that is it.

    In the case that you are backing up your entire database for security reasons only, choose backup database and not backup company.  Backup company is only for when you want to then import the file again under a new name.

    You should save this information to some kind of removeable memory device, like disketts or tape.

  • When

    • This is up to you.  I suggest at least once a day if you work intensively with this program.
